# Customer Segmentation using KMeans Clustering

## Project Overview

This project implements a complete machine learning pipeline for customer segmentation using KMeans clustering followed by a classification model to categorize new customers. The objective of the project is to analyze customer behavior and group customers into meaningful segments that can help businesses design targeted marketing strategies and improve resource allocation.

The project also includes a Streamlit application where users can input customer features and receive an immediate prediction of the customer segment.

## Methodology

### 1. Data Engineering and Preprocessing

The project begins with preparing the dataset and ensuring the data is suitable for modeling.

**Data Collection**  
The dataset contains customer features such as Age, Annual Income and Spending Score.

**Data Cleaning**  
Missing values were handled and the dataset was checked for possible outliers.

**Feature Scaling**  
Since KMeans clustering relies on distance calculations, feature scaling was applied using normalization or standardization. This ensures that all features contribute equally to the clustering process.

---

### 2. Model Training

#### Clustering (Unsupervised Learning)

The Elbow Method was used to determine the optimal number of clusters. Based on the analysis, five clusters were selected.

The KMeans algorithm was then trained to group customers into five distinct segments based on their characteristics.

Cluster visualizations were created using Matplotlib to better understand the distribution of customer groups.

#### Classification (Supervised Learning)

The cluster labels generated by the KMeans algorithm were used as the target variable for training a classification model.

A Decision Tree Classifier was trained to map customer input features to the final customer segment.

Both the KMeans model and the Decision Tree model were saved using serialized files for later use.

---

### 3. Application Integration

A Streamlit application was developed to allow users to interact with the trained models.

Users can enter customer information through the interface, and the application predicts the customer segment in real time. The application loads the saved models and uses them to perform predictions without retraining the models.
